# 环境安装与运行

本节给出当前仓库的环境准备方式、实验命令入口和运行注意事项，覆盖 `notes/00-13` 对应的公开实验。

## 环境准备

`conda` 环境命令如下：

```bash
conda env create -f environment.yml
conda activate ReinforcementLearning
```

纯 `pip` 环境建议先安装 `swig`，再安装项目依赖：

```bash
pip install swig
pip install -r requirements.txt
```

当前依赖主项如下：

- `python`
- `numpy`
- `matplotlib`
- `gymnasium[box2d]`
- `tqdm`
- `torch`
- `swig`

## 运行说明

- 当前仓库覆盖动态规划、表格型无模型方法、深度值函数方法、策略梯度方法和连续动作控制。
- `DQN`、`REINFORCE`、`Actor-Critic`、`PPO` 与 `SAC` 脚本会自动选择设备：`cuda` 可用时使用 `cuda`，否则回退 `cpu`。
- `LunarLander-v3` 依赖 `Box2D`。`environment.yml` 已包含 `swig` 与 `gymnasium[box2d]`，纯 `pip` 环境需先执行 `pip install swig`。
- 默认配置以教学可复现为优先，深度方法在 CPU 环境下也可完整运行，但耗时会明显高于表格法。

## 常见问题

### `gymnasium` 或 `torch` 导入失败

终端环境未切换到项目环境时会出现该问题，可执行：

```bash
conda activate ReinforcementLearning
```

### `LunarLander` 提示缺少 `Box2D`

优先检查环境是否按照 `environment.yml` 创建。纯 `pip` 环境可执行：

```bash
pip install swig
pip install -r requirements.txt
```

### 图像没有显示或保存失败

优先检查 `matplotlib` 是否安装，并确认当前目录存在写权限。
