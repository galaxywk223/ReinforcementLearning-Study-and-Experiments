# 环境安装与运行

本节给出当前仓库的环境准备方式、实验命令入口和运行注意事项，覆盖 `notes/00-07` 对应的公开实验。

## 环境准备

`conda` 环境命令如下：

```bash
conda env create -f environment.yml
conda activate ReinforcementLearning
```

`pip` 环境命令如下：

```bash
pip install -r requirements.txt
```

当前依赖主项如下：

- `python`
- `numpy`
- `matplotlib`
- `gymnasium`
- `tqdm`
- `torch`

## 运行命令

### `01` FrozenLake / Q-Learning

```bash
cd experiments/01-frozenlake-tabular-q
python train.py --episodes 4000 --render-final-policy
python trace_q_updates.py --episodes 6
```

### `02` CliffWalking / SARSA

```bash
cd experiments/02-cliffwalking-tabular-sarsa
python train.py --episodes 800 --render-final-policy
python compare_sarsa_q_learning.py --episodes 800
```

### `03` Blackjack / Monte Carlo

```bash
cd experiments/03-blackjack-monte-carlo
python train.py --episodes 200000 --render-final-policy
python trace_mc_updates.py --episodes 3
```

### `04` CliffWalking / n-step SARSA

```bash
cd experiments/04-cliffwalking-n-step-sarsa
python train.py --episodes 800 --n-step 4 --render-final-policy
python compare_one_step_n_step_sarsa.py --episodes 800 --n-step 4
python trace_n_step_updates.py --episodes 3 --n-step 4
```

### `05` CartPole / DQN

```bash
cd experiments/05-cartpole-dqn
python train.py --episodes 400 --print-eval-rollout
python trace_dqn_updates.py
```

## 运行说明

- 当前仓库同时包含表格方法和 `CartPole` 上的 `DQN`。
- `DQN` 脚本会自动选择设备：`cuda` 可用时使用 `cuda`，否则回退 `cpu`。
- 默认配置以教学可复现为优先，CPU 环境也可完整运行全部实验。

## 常见问题

### `gymnasium` 导入失败

终端环境未切换到项目环境时会出现该问题，可执行：

```bash
conda activate ReinforcementLearning
```

### 图像没有显示或保存失败

优先检查 `matplotlib` 是否安装，并确认当前目录存在写权限。
